# Working with Codex

## Mission
Describe brevemente el propósito del proyecto y el rol de Codex.

## Core Principles
- No modificar la arquitectura sin autorización.
- Priorizar componentes reutilizables.
- Evitar duplicación de contenido.
- Mantener compatibilidad con GitHub Pages.

## Repository Rules
- Respetar la estructura de carpetas.
- Documentar cambios relevantes.
- No eliminar archivos existentes sin justificación.

## Development Guidelines
- Crear soluciones simples antes que complejas.
- Preferir configuraciones sobre código personalizado cuando sea posible.
- Mantener consistencia en nombres y organización.

## Pull Request Checklist
- ¿Respeta la arquitectura?
- ¿Rompe compatibilidad?
- ¿Actualizó la documentación si era necesario?
